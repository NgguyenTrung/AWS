### Week 4 Objectives:

* Learn about data storage services on AWS.
* Understand the role and operation of Amazon Simple Storage Service.
* Differentiate between Amazon S3 storage classes and select the appropriate class for different use cases.
* Learn how to deploy a static website, configure CORS, and control access permissions on Amazon S3.
* Become familiar with Amazon S3 Glacier, AWS Snow Family, AWS Storage Gateway, and AWS Backup.

### Week 4 Achievements:

* I understood the role of Amazon S3 in providing object storage on AWS.

* I learned about the fundamental components of Amazon S3, including:

  * Bucket
  * Object
  * Object Key
  * Access Point

* I differentiated between Amazon S3 storage classes and understood the appropriate use case for each class.

* I understood how to use Amazon S3 to host a static website and configure CORS for a web application.

* I learned about the following methods for controlling access to Amazon S3:

  * IAM Policy
  * Bucket Policy
  * Block Public Access
  * Object Ownership

* I understood how Amazon S3 Glacier supports low-cost, long-term data storage.

* I learned about the role of AWS Snow Family in transferring and migrating large volumes of data to AWS.

* I understood how AWS Storage Gateway connects on-premises storage systems to the AWS Cloud.

* I learned how AWS Backup centrally manages data backup and restoration.

* I understood the AWS Shared Responsibility Model and differentiated between the security responsibilities of AWS and those of its customers.

* I learned about the role of AWS IAM in identity and access management.

* I understood how Amazon Cognito is used to authenticate and manage application users.

* I learned how AWS Organizations supports the centralized management of multiple AWS accounts.
