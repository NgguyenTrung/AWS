### Week 2 Objectives:

* Learn the fundamental concepts of networking on AWS.
* Understand the role and main components of Amazon Virtual Private Cloud.
* Learn about security mechanisms and connectivity between multiple VPCs.
* Differentiate between network connectivity methods such as VPN and AWS Direct Connect.
* Become familiar with Amazon Route 53 and AWS Transit Gateway.
* Practise deploying and managing network resources through AWS labs.

### Week 2 Achievements:

* I understood the role of Amazon Virtual Private Cloud in building a private network environment on AWS.

* I learned the basic components of a VPC, including:

  * CIDR Block
  * Public Subnet
  * Private Subnet
  * Route Table
  * Internet Gateway
  * Security Group
  * Network ACL

* I understood the differences between Public Subnets and Private Subnets.

* I learned how Security Groups and Network ACLs are used to control inbound and outbound traffic for AWS resources.

* I learned about connectivity methods between multiple VPCs and between on-premises systems and AWS, including:

  * VPC Peering
  * AWS Site-to-Site VPN
  * AWS Direct Connect
  * AWS Transit Gateway

* I understood the role of Elastic Load Balancing in distributing incoming traffic across multiple resources.

* I practised creating a VPC, subnets, route tables, and an Internet Gateway, as well as configuring security through the **Networking Essentials with Amazon VPC** lab.

* I understood how Amazon Route 53 supports DNS management and DNS connectivity in a hybrid environment.

* I understood how AWS Transit Gateway supports centralized connectivity and management for multiple VPCs.

* The knowledge gained in Week 2 provided me with a foundation in AWS networking, enabling me to continue learning about cloud architectures and services in the following weeks.
